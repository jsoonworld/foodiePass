# FoodiePass Frontend - UI/UX Requirements

> **목적**: UI/UX 요구사항 및 디자인 가이드라인

---

## 핵심 원칙

### 1. A/B 테스트 무결성 유지
- Control vs Treatment UI는 **정확히** 스펙대로 구현
- 사용자가 그룹을 인지하거나 변경할 수 없어야 함
- 두 UI의 차이는 **오직** 사진/설명 유무

### 2. 모바일 우선 (Mobile-First)
- 여행객은 주로 스마트폰 사용
- 터치 인터랙션 최적화
- 네트워크 속도 고려 (이미지 최적화)

### 3. 단순성 (Simplicity)
- 복잡한 네비게이션 불필요
- 직관적인 플로우: 업로드 → 결과 → 설문
- 불필요한 정보 제거

## Control UI (F-07)

### 레이아웃
```
┌─────────────────────────┐
│ Margherita Pizza        │ ← 원어 메뉴명
│ 마르게리타 피자           │ ← 번역명
│ $15.00 (₩20,000)       │ ← 가격
└─────────────────────────┘
```

### 스타일링
- 배경: 흰색
- 텍스트: 왼쪽 정렬
- 간격: 각 아이템 사이 16px
- 폰트:
  - 원어: 16px, bold
  - 번역: 14px, regular
  - 가격: 14px, semibold

### 제약사항
- ❌ 사진 없음
- ❌ 설명 없음
- ✅ 메뉴명 + 가격만

## Treatment UI (F-06)

### 레이아웃
```
┌─────────────────────────┐
│  [음식 사진]             │ ← 이미지 (4:3 비율)
│                         │
│ Margherita Pizza        │ ← 원어 메뉴명
│ 마르게리타 피자           │ ← 번역명
│ 토마토 소스, 모짜렐라... │ ← 설명 (2-3줄)
│ $15.00 (₩20,000)       │ ← 가격
└─────────────────────────┘
```

### 스타일링
- 배경: 흰색 카드 (그림자 효과)
- 이미지: 상단, 4:3 비율, 라운드 코너
- 텍스트: 왼쪽 정렬
- 간격: 카드 사이 16px
- 폰트:
  - 원어: 18px, bold
  - 번역: 16px, regular
  - 설명: 14px, light gray
  - 가격: 16px, semibold

### 제약사항
- ✅ 사진 포함 (없으면 placeholder)
- ✅ 설명 포함
- ✅ 메뉴명 + 가격

## 설문 UI (F-09)

### 레이아웃
```
┌─────────────────────────┐
│ 이 정보만으로 확신을 갖고  │
│ 주문할 수 있습니까?       │
│                         │
│  ┌─────┐   ┌─────┐    │
│  │ Yes │   │ No  │    │
│  └─────┘   └─────┘    │
└─────────────────────────┘
```

### 스타일링
- 배경: 반투명 오버레이 (rgba(0,0,0,0.5))
- 모달: 흰색, 중앙 정렬, 라운드 코너
- 질문: 18px, bold, 중앙 정렬
- 버튼: 크기 동일, 충분한 터치 영역 (최소 48x48px)

### 동작
- 5초 후 자동 표시
- Yes/No 클릭 → API 제출 → "감사합니다!" → 모달 닫기
- 백드롭 클릭 시 무시 (반드시 응답해야 함)

## 공통 UI 요소

### 업로더 (F-01)

**이미지 업로드 영역**:
- 드래그앤드롭 지원
- 클릭하여 파일 선택 가능
- 미리보기 표시
- 허용 포맷: JPG, PNG, HEIC
- 최대 크기: 10MB

**언어/화폐 선택**:
- 드롭다운 형식
- 기본값: 한국어 (ko), KRW
- 지원 언어: 한국어, 영어, 일본어, 중국어 등
- 지원 화폐: KRW, USD, JPY, CNY 등

**버튼**:
- "Scan Menu" 버튼 (primary)
- 비활성 상태: 이미지 미선택 시
- 로딩 상태: "Processing..." 표시

## 로딩 인디케이터

### 스타일
- 중앙 정렬 스피너
- 메시지: "메뉴를 분석하는 중..."
- 예상 시간 표시: "보통 5초 정도 걸립니다."

### 위치
- 전체 화면 오버레이
- 반투명 배경

---

## 에러 메시지

### 스타일
- 빨간색 배경 (light red)
- 아이콘: ⚠️
- 메시지: 명확하고 간결
- 액션 버튼: "다시 시도" (retry)

### 위치
- 화면 상단 (toast) 또는 인라인

## 반응형 브레이크포인트

| 모바일 | < 640px | 1열 |
| 태블릿 | 640px - 1024px | 2열 |
| 데스크톱 | > 1024px | 2-3열 |

---

## 접근성 (Accessibility)

### 필수 요구사항
- 키보드 네비게이션 지원
- Alt 텍스트 (이미지)
- ARIA 레이블 (버튼, 폼)
- 색상 대비 (WCAG AA 이상)
- 터치 영역 최소 48x48px
